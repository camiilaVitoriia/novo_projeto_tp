Mas antes de tudo... O que é GitHub?

Em resumo, trata-se de um serviço baseado em nuvem que hospeda um sistema de controle de versão chamado Git. Assim, ele possibilita que os desenvolvedores colaborem e façam mudanças em projetos compartilhados enquanto mantêm um registro detalhado do seu progresso.

Entenda um pouco mais sobre controle de versão

Essa ferramenta auxilia programadores em seus projetos. Para isso, os desenvolvedores criam um novo projeto eles continuam criando atualizações no código base. Mesmo depois de o projeto ser lançado é comum a atualização de versões, correção de bugs, adição de novas ferramentas, etc. O sistema de controle de versão ajuda a acompanhar as mudanças feitas no código base. E mais, ele também registra quem efetuou a mudança e permite a restauração do código removido ou modificado.

Você sabe qual é o sistema de controle de versão mais usado hoje em dia?

Claro que é o git! sendo ele um projeto de código aberto maduro e com manutenção ativa desenvolvido em 2005 por Linus Torvalds. Um número impressionante de projetos de software depende do Git para controle de versão, incluindo projetos comerciais e de código-fonte aberto. Ou seja, o preferidinho entre todos.

Tendo uma arquitetura distribuída, o Git é um exemplo de portanto, Sistema de Controle de Versão Distribuído. Em vez de ter apenas um único local para o histórico completo da versão do software, como é comum em sistemas de controle de versão outrora populares como CVS ou Subversion, no Git, a cópia de trabalho de todo desenvolvedor do código também é um repositório que pode conter o histórico completo de todas as alterações. Além de ser distribuído, o Git foi projetado com desempenho, segurança e flexibilidade em mente.
